1.我要搭建一个公司的AI问答系统，
2.知识库包括公司规章制度（word文档）、遇到的问题（word文档，多个类型）、保存了的数据集（是否训练、名称等多个字段）
3.用户端包括选择知识库合集上传文档，增加数据集记录
4.管理端包括用户管理，知识库管理，数据集记录管理

- normal 简单模式，上下文对话三次,只是对话，不做额外工作
- agent 智能模式，根据用户问题，从知识库中提取相关信息，生成回答
  - 实际流程：
    - 1）输入：用户问题 + 上下文 + 是否查询知识库 + 是否联网搜索
    - 2）意图识别与动作选择：
      - 模型先做意图识别，判断需要“仅对话/查知识库/联网搜索/组合”
      - 如果要查询知识库：先给出可用知识库列表，让模型选择“查哪个知识库（collection）”，再做检索
      - 如果要联网搜索：执行搜索，并根据搜索结果列表选择合适工具（如联网搜索、URL读取）
      - 根据工具列表让模型选择合适的工具
    - 3）第一次总结与自检：
      - 总结 response 并判断是否与问题相关
      - 若不相关：自动换策略再试一次：让模型选择更改关键词重新搜索，或向用户提问来总结关键词再搜索
    - 4）第二次总结：
      - 再次总结 response
      - 若仍不理想：也要给出当前能得到的总结与依据并返回
