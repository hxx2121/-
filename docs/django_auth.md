# django_auth的APP的要求
## 代码实现说明
- 使用Django默认的User表
- 在models.py增加一个用户信息表，用于存储用户额外的信息
- utils.py 定义一个装饰器，用于判断用户是否登录，以及用户是否是管理员
- 统一返回格式django_main\R.py
- 使用序列化器，并且视图函数继承GenericAPIView
- 需求之外的不要写，如果有额外必须的，请和我确认再写

## 功能模块(接口路由)
- 用户注册
- 用户登录
- 用户退出登录
- 用户修改个人信息（用户名、密码、角色）
- 管理员可以查看所有用户信息
- 管理员可以删除用户
- 管理员可以修改用户信息（用户名、密码、角色）
